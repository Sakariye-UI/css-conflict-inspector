How to use this tool Features, FAQs & error guide
✨ Features ❓ FAQs 🔴 Errors
✨ Features
🔍 Scan Page
Inspect the active tab for anything breaking a Klaviyo component
  • Detects display:none, visibility:hidden, opacity:0, overflow:hidden, max-height:0, pointer-events:none and transform conflicts
  • Traces every issue back to the exact stylesheet, selector, and line causing it
  • Shows a confidence score (1–5) so you know which issue to fix first
  • Recognises Shopify Dawn, Elementor, Divi, Tailwind, Bootstrap and more — names the theme and suggests the exact fix
  • Detects z-index stacking issues, script load problems, localStorage blocks, and consent manager (CMP) conflicts
  • Generates a ready-to-send customer message with the issue and fix pre-written
⚠️ Won't work if: Klaviyo's script tag isn't on the page, the page is a file:// URL, or a strict CSP blocks content scripts
Pick Element
Click any element on the page to inspect it directly
  • Hover over any element — it highlights in purple
  • If you click a Klaviyo component (form, embed, flyout), the tool runs the full CSS conflict scan on that component and shows the same detailed output as Scan Page
  • If you click a generic element, the tool runs a CSS visibility check on that element and its ancestors, showing any display, visibility, opacity, overflow, or pointer-events issues found
  • Useful when you already know which element looks broken and want to skip the full scan
⚠️ Won't work if: The element is inside a cross-origin iframe
📋 Manual Check
Check a specific form by its ID
  • Enter a Form ID (from the dashboard URL or data-testid) and click Check
  • Optionally enter a Variation ID to test a specific A/B variant
  • Runs the same CSS conflict detection as Scan Page but targets that one form
🚀 Force Open & Force Open (Ignore Rules)
Open a form that isn't showing — bypass targeting or triggers
  • Force Open — signals Klaviyo to open the form as if the trigger fired. Respects geo, cookie, and targeting rules
  • Force Open (Ignore Rules) — clears Klaviyo's localStorage, reloads the page, and forces the form open regardless of targeting. Use this when the form never appears even for new visitors
  • If a Variation ID is entered, only that A/B variant is shown
⚠️ Won't work if: The Klaviyo onsite script isn't on the page, or the account has strict geo restrictions that block the API response entirely
🧪 Test Submit
Fill and submit a form with your test profile to confirm the API call succeeds
  • Automatically fills email, phone, first name, last name, and date of birth fields using the test profile set in Settings
  • Name defaults to Klaviyo Test — change it in Settings under "Test Name"
  • Date of birth defaults to 01/01/2000 — update it in Settings. The tool recognises birthday, date_of_birth, dob, birthdate, and MM/DD/YYYY fields automatically
  • Clicks the form's submit button and intercepts the Klaviyo API call
  • Reports whether the subscription went through (success) or failed (with the error reason)
  • Useful for confirming a form submits correctly end-to-end after fixing a CSS conflict
⚠️ Won't work if: The form has multi-step flows the tool can't navigate, or the site blocks synthetic events with isTrusted checks on embedded forms
🌐 Site Scanner
See every Klaviyo form on a merchant's account
  • Fetches all active forms for the account from Klaviyo's API using the company ID found on the page
  • Shows form name, ID, type (modal / flyout / embed), and which pages it targets
  • Flags snippet-placed forms — those embedded directly in the page HTML rather than shown via Klaviyo's trigger system
  • Force Open and Test Submit are available for each form directly from the scanner
⚠️ Won't work if: Klaviyo's script tag isn't on the page, the account has no active forms, or geo-IP filtering means forms for other regions aren't returned
🕒 Scan History
Review recent scans and HAR captures
  • Recent Scans — stores the last 5 page scans. Each entry shows the URL, timestamp, issue count, and the full scan results. Click any entry to re-open it
  • HAR Captures — stores the last 10 HAR recordings. Each entry shows the URL, timestamp, and number of network entries captured. Download the file from here at any time
  • Both tabs persist across popup opens and browser restarts until you clear the extension's storage
  • Scans are saved automatically every time you run Scan Page, Manual Check, or Pick Element on a Klaviyo component
📱 Mobile Preview
Emulate a real iPhone or Android device to see how the form renders on mobile
  • Uses Chrome's built-in device emulation to set exact screen dimensions, pixel ratio, and User-Agent
  • Choose iOS (iPhone 14, 390×844, DPR 3) or Android (Pixel 7, 412×915, DPR 2.625)
  • Emulation stays active even after closing the panel — toggle it off or close the tab to stop
  • Because the User-Agent changes, Klaviyo may serve different CSS or variants — scans while mobile is active reflect what a real mobile visitor would see
⚠️ Won't work if: Chrome DevTools is open on the same tab — Chrome only allows one debugger at a time. Close DevTools first, then activate Mobile Preview
🚩 Report
Send feedback or log a new CSS conflict pattern directly from the tool
  • Select an event from a recent scan to include its context automatically
  • Choose a report type: New Error (pattern not detected), False Positive (flagged issue that isn't actually a problem), or Other Feedback
  • Add a comment describing what you saw, then submit — it goes straight to the development queue
👁️ Preview Fix
Temporarily override a conflicting CSS property to see if the form appears
  • Appears next to detected issues after a scan
  • Click to disable the conflicting property live on the page — if the form appears, you've found the cause
  • Click Restore to put it back
⚠️ Not available for zero-dimension issues — the fix depends on what's setting the size, which varies per theme
📡 HAR File
Request a network trace from a customer or record one yourself
  • Record Your Session — starts a live HAR capture on the active tab. Reproduce the issue, then click Stop & Export to save. The file is copied to your clipboard and stored under Scan History → HAR Captures for later reference
  • Customer Macro — a pre-written message with step-by-step HAR instructions for Chrome, Firefox, and Safari. Click Copy Macro and paste directly into Zendesk — screenshots for each browser are embedded and appear inline in the ticket automatically
  • HAR captures are saved per session and accessible any time under Scan History → HAR Captures
⚠️ Won't work if: Chrome DevTools is already open on the tab — Chrome only allows one debugger at a time. Close DevTools first, then start recording
❓ FAQs
Why does it say "0 forms found" after scanning?
Klaviyo loads form HTML asynchronously after its SDK initialises. If you scan immediately after page load the form may not be in the DOM yet. The tool retries once automatically — if it still shows nothing, wait a few seconds and scan again.
Why does the Preview Fix button not appear on some issues?
Zero-dimension issues (the element has no width or height) don't have a single property to override — the fix depends on what's setting the size in that theme. Use the source file and selector info shown to find and edit it manually.
Does using this tool affect real customers?
No. Scan Page and Pick Element are read-only. Force Open only affects the tab you're on. Test Submit uses your test email/phone from Settings — it does create a real profile for that address, which is exactly what you want when testing. Use a dedicated Klaviyo test email and phone number so the profile stays easy to identify.
What does the confidence score mean?
It's a 1–5 rating of how likely this CSS issue is the actual cause of the form being hidden. Score 5 = almost certainly the cause (e.g. display:none !important on the form root). Score 1–2 = a potential factor but probably not the only cause. Start with the highest score.
Why does Test Submit need two tries sometimes?
On SMS forms, Klaviyo's country picker updates its React state asynchronously. The tool detects this and retries automatically — filling email and phone on the first attempt, then re-filling if the country picker wasn't ready yet. Names are only filled on the first step to avoid duplicating entries on multi-step forms.
The scan found issues but the form still looks fine — why?
Some CSS properties flagged are on parent containers, not the form itself. If the form is a position:fixed flyout it can escape overflow clipping. The confidence score helps — low-confidence warnings may not actually be hiding the form.
Can I change the name used when Test Submit fills name fields?
Yes. Open Settings and scroll to the "Test Name" section. Enter the first and last name you want used — defaults are Klaviyo (first) and Test (last). Leave a field blank to skip filling that name on the form.
Does Mobile Preview affect how the scan results look?
Yes — when Mobile Preview is active, Klaviyo may serve different form variants or CSS based on the spoofed User-Agent. This means scan results reflect what a real mobile visitor would see, which can surface mobile-specific CSS conflicts that don't appear on desktop.
What does "snippet-placed" mean in Site Scanner?
Snippet-placed forms are embedded directly in the page HTML using Klaviyo's embed code, rather than being triggered by Klaviyo's popup system. They don't respond to Force Open. If a snippet-placed form isn't showing, the issue is in the page HTML or the CSS around the embed container — use Scan Page or Pick Element to inspect it directly.
Where do my recorded HAR captures go?
They're saved automatically to Scan History → HAR Captures. Each entry shows the URL, timestamp, and number of network entries recorded. They persist until you clear the extension's storage.
Can I record a HAR file and keep working on the same tab?
Yes. Once recording starts you can close the extension panel and use the tab normally. Reopen the panel and go to the HAR File section, then click Stop & Export when you're ready. The recording continues in the background.
🔴 Error Reference
⚠️ Klaviyo not detected on this page
The Klaviyo onsite script (company_id=…) wasn't found in the page's script tags. Navigate to the merchant's live site and make sure the Klaviyo snippet is installed. Won't work on the Klaviyo dashboard itself.
⚠️ Form not found in DOM
The form ID entered doesn't match any element in the page's DOM. Either the form hasn't loaded yet (try Force Open first), the ID is from the wrong account, or Klaviyo rendered it with a different DOM ID than the dashboard shows.
⚠️ Fields filled but no API call detected
The tool filled the form fields and clicked submit, but didn't detect a call to Klaviyo's subscription API within 30 seconds. This usually means the form's own validation blocked the submission (e.g. the email was already subscribed, or a CAPTCHA fired). Try submitting the form manually while the tool watches — it will detect and report the API result.
⚠️ Could not fetch forms from Klaviyo's API
The Site Scanner couldn't load the account's form list. This can happen if the Klaviyo API is temporarily unavailable, the account has strict geo restrictions, or the company ID on the page doesn't match an active Klaviyo account.
⚠️ Signal sent but form was not detected after reload
Force Open (Ignore Rules) reloaded the page and sent the open signal, but the form didn't appear in the DOM. The most common causes: the account has geo-IP targeting that excludes your current location, the form is paused or archived, or the Klaviyo SDK failed to load after reload.
⚠️ This number is invalid
Klaviyo rejected the phone number during Test Submit. This usually means the country code doesn't match the number format (e.g. a US number with a UK country code set). Check the phone number in Settings — it should be in full E.164 format, e.g. +14155552671.
⚠️ Could not start Mobile Preview — close DevTools first
Chrome only allows one debugger connection per tab. If Chrome DevTools is open on the active tab, Mobile Preview can't attach. Close DevTools (press F12 or ⌥⌘I), then try activating Mobile Preview again.
⚠️ Could not start HAR recording — make sure DevTools is closed on this tab
HAR recording uses Chrome's debugger API, which conflicts with open DevTools. Close DevTools on the active tab (F12 or ⌥⌘I) and try again. If Mobile Preview is also active, deactivate it first — both features share the same debugger connection.
⚠️ Recording stopped — no data captured
The HAR export returned empty. This usually means the recording started after the network activity you wanted to capture. Start the recording first, then trigger the issue on the page, then stop. If the page needs a full reload to reproduce the issue, begin recording before refreshing.
